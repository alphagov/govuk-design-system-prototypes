Overview

When in what ways to display tabular data.

When to use this pattern

Tables should only be used to present data.

When NOT to use this pattern

Don’t use tables for cosmetic changes to layout, eg to present a list because you think it looks better that way.

How it works

Below are different was to implement tables to display data across different screen sizes.

Add appropriate scope=“col” to thead items and scope=“row” and each th element in mobile view.

Responsive tables

Use this pattern when there is not enough space on smaller screen to fit traditional table layout. This pattern requires adding data-* attribute to each row cell that mimics the column header text. On smaller screens this text is then displays before the cell value, making information easier to read.

i Doctoral loans 0 -5 -5 -10 -10
ii Part-time maintenance loans 0 75 130 95 65
iii Further Education maintenance loans 0 0 -15 -40 -70
Total policy decisions 0 70 110 45 -15

Responsive tables - Javascript solution

This is a Javascript solution that turns a standard table into a responsive table using the above describedsolution without the need for manually adding data-* attributes

2017-18 2018-19 2019-20 2020-21 2021-22
i Doctoral loans 0 -5 -5 -10 -10
ii Part-time maintenance loans 0 75 130 95 65
iii Further Education maintenance loans 0 0 -15 -40 -70
Total policy decisions 0 70 110 45 -15 
var headertext = [],
headers = document.querySelectorAll(".js-make-table-responsive th"),
tablerows = document.querySelectorAll(".js-make-table-responsive th"),
tablebody = document.querySelector(".js-make-table-responsive tbody");

for(var i = 0; i < headers.length; i++) {
  var current = headers[i];
  headertext.push(current.textContent.replace(/\r?\n|\r/,""));
}
for (var i = 0, row; row = tablebody.rows[i]; i++) {
  for (var j = 0, col; col = row.cells[j]; j++) {
    col.setAttribute("data-th", headertext[j]);
  }
}
