Navigation

BreadcrumbsApproved

Last reviewed:
30 March 2017
Edit this page on:
Github

Overview

Help users orientate themselves and navigate pages within a hierarchical structure.

  1. Home
  2. Level one
  3. Level two
  4. Level three

When to use this component

Breadcrumbs are a form of navigation that convey the hierarchical relationship between pages.

They are also used for orientation. If a user lands on any page they should be able to understand where they currently are in the hierarchy of a website.

When not to use this component

Breadcrumbs are not appropriate for use in a flat website structure, or to convey progress through a linear journey or transaction. We recommend using back button for this.

Their use should also be considered against other navigational devices on the page. Breadcrumbs are not always necessary where there is permanently visible navigation, such as a sidebar.

How it works

Standard example

Your breadcrumb should include the user’s current page, however it should be visually different from the other links within the breadcrumb.

  1. Home
  2. Level one
  3. Level two
  4. Level three 
<div class="breadcrumbs">
  <ol>

    <li class="">
        <a class="" aria-current="false" href="/">Home</a>
    </li>

    <li class="">
        <a class="" aria-current="false" href="#">Level one</a>
    </li>

    <li class="">
        <a class="" aria-current="false" href="#">Level two</a>
    </li>

    <li class="breadcrumbs__current-page">
        <a class="" aria-current="page" href="#content">Level three</a>
    </li>
  </ol></div>

Mobile varient example Experimental

When displayed on a small screen device the visual relationship between pages in the breadcrumb are lost due to content wrapping over multiple lines. It also pushes the main content further down the page.

This component has a CSS modifier breadcrumbs--collapse which collapses the breadcrumb into a single link, similar to the back button.

This reveals the parent of the current page so the user can travel up through the hierarchy of the site or section one level at a time.

  1. Home
  2. Level one
  3. Level two
  4. Level three 
<div class="breadcrumbs breadcrumbs--collapse">
  <ol>

    <li class="">
        <a class="" aria-current="false" href="/">Home</a>
    </li>

    <li class="">
        <a class="" aria-current="false" href="#">Level one</a>
    </li>

    <li class="breadcrumbs__parent">
        <a class="" aria-current="false" href="#">Level two</a>
    </li>

    <li class="breadcrumbs__current-page">
        <a class="" aria-current="page" href="#content">Level three</a>
    </li>
  </ol></div>

Navigation markup

If the breadcrumb is the only form of navigation on the page then it should be marked up using <nav> in place of a <div>. This is to limit confusion screen reader users may face with multiple navigational components on the page.

Research on this component

Many iterations of this component have been tested across Gov.uk. Some observations from research are

  • People do use breadcrumbs to navigate and have a strong mental model of how breadcrumbs work (they show how content is organised)
  • There isn’t a common way they are used for orientation
  • People used them to “go home” when they feel too deep in the website
  • Some users understood that they represent hierarchy of the content, however some users expect breadcrumbs to show the path they have personally taken to reach that specific piece of content
  • People used them to go back one step, hence often compared to a back button - allowing to navigate in a smaller section of the website
    • When the breadcrumbs were tested without the “Current page” users were observed jumping back two levels
  • Some people didn’t interact with them because they didn’t notice them and generally prefer to use the browser back button

Very little research has been done on the mobile presentation of this component.
