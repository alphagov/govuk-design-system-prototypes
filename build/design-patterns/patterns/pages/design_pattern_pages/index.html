A design pattern is a repeatable way of solving a common design problem.

Design patterns consist of guidance, examples and often reusable code.

Design pattern meta data

Every design pattern has meta data that describes the pattern and it’s properties. The meta data is coded as YAML Frontmatter at the top of the markdown file for each pattern.

This meta data is used to build the navigation for GOV.UK Design Patterns, so it’s important that it’s correct. The table below describes each data item and the values it can take:

Variable Description Possible values
title The name of the pattern Any
aliases Other common names or keywords associated with the pattern Comma separated list
section The section the pattern appears in the navigation Styles, Components, Pages, Tasks or Services
status The status of the pattern Backlog, Work-in-progress, Beta or Approved
department Which departments the pattern is relevant for Use ‘All’ for global patterns, otherwise use the name of the department
priority The relative importance of a backlog pattern Number - low means high priority
